# 课程目标达成度报告工具

## 1. 项目简介

本项目用于从课程基础信息、学生过程性成绩、试卷分值映射和试卷题目得分明细中，自动计算课程目标达成度，并生成课程目标结果表、达成度报告分析底表、Word 报告草稿和持续改进报告图表。

当前验证课程为：

```text
input_courses/材料成型技术_2025-2026-1/
```

## 2. 当前支持的功能

- 自动读取课程目录下的标准输入 Excel 文件。
- 清洗输入表列名中的空格、换行和全角空格。
- 兼容不同格式的课程目标编号，例如 `1`、`目标1`、`CO1`、`课程目标1`。
- 按课程目标分别计算过程性成绩、期末成绩和综合达成度。
- 根据 `03_试卷分值对应表.xlsx` 的题号-课程目标映射汇总期末试卷得分。
- 输出 `05_课程目标结果表.xlsx`。
- 输出多 sheet 的 `06_达成度报告分析底表.xlsx`。
- 生成 `07_达成度报告草稿.docx`。
- 使用 matplotlib 生成 3 张持续改进报告图表，并插入 Word 报告。
- 可选启用原始成绩表预处理，将 `raw_inputs/` 中的原始成绩表转换为标准 `02` 和 `04` 输入表。
- 支持试卷型课程和非试卷型课程：
  - 课程目录中同时存在 `03_试卷分值对应表.xlsx` 和 `04_试卷题目得分长表.xlsx` 时，走试卷型流程。
  - 课程目录中不存在 03/04 时，走纯过程评价流程，仅依赖 `01_课程主数据表.xlsx` 和 `02_学生成绩表*.xlsx`。
- 支持 `DEBUG` 开关，控制 columns、shape、唯一值、head 等调试输出。

## 4. 依赖安装方式

建议在项目根目录 `course_report_tool/` 下安装依赖：

```bash
python -m pip install -r requirement.txt
```

当前依赖包括：

```text
pandas
openpyxl
python-docx
matplotlib
```

## 5. 运行命令

在 `course_report_tool/` 目录下运行：

```bash
python -m src.main
```

如需先从 `raw_inputs/` 中的教师原始成绩表生成标准 `02_学生成绩表.xlsx`，可运行：

```bash
python -m src.main --course 材料成型技术_2025-2026-1 --prepare-scores
```

该命令会同步生成：

```text
input_courses/<课程目录>/02_学生成绩表.xlsx
input_courses/<课程目录>/02_学生成绩表_生成核查报告.xlsx
```

程序会自动选择 `input_courses/` 下最近修改的课程目录，并将结果写入：

```text
output/<课程目录名>/
```

也可以通过环境变量指定课程目录：

```powershell
$env:COURSE_DIR_NAME="科研训练计划_2025-2026-2"
python -m src.main
Remove-Item Env:COURSE_DIR_NAME
```

如需启用或关闭原始成绩表预处理，可修改 `src/main.py`：

```python
PREPROCESS_RAW_INPUTS = True
```

当 `PREPROCESS_RAW_INPUTS=True` 时，程序会先读取 `raw_inputs/`，生成标准 `02_学生成绩表.xlsx` 和 `04_试卷题目得分长表.xlsx`，再执行原有达成度计算流程。

当 `PREPROCESS_RAW_INPUTS=False` 时，程序直接使用 `input_courses/<课程目录>/` 下已有标准输入表。

非试卷型课程不需要 03 和 04 文件。程序会自动识别课程目录中是否存在 03/04，并选择对应计算流程。

## 6. 输入文件说明

### 01_课程主数据表.xlsx

课程基本信息与课程目标信息。核心字段：

- `课程名称`
- `课程代码`
- `开课学期`
- `课程目标编号`
- `课程目标描述`
- `课堂表现权重`
- `课程作业权重`
- `实验报告权重`
- `实验操作权重`
- `期末权重`

### 02_学生成绩表.xlsx

学生过程性成绩表。核心字段：

- `学号`
- `姓名`
- `课程目标编号`
- `平时作业`
- `实验`
- `实验报告`
- `课堂表现`
- `所属课程`
- `所属学期`

如果由 `raw_inputs/副本材料成型技术基础-课程目标达成情况评价表.xlsx` 预处理生成，输出字段包括：

- `学号`
- `姓名`
- `课程目标编号`
- `课程作业`
- `实验操作`
- `实验报告`
- `课堂表现`
- `期末考试成绩`
- `原表达成度`
- `所属课程`
- `所属学期`

说明：`loaders.py` 会将 `课程作业` 兼容为 `平时作业`，将 `实验操作` 兼容为 `实验`，以适配现有计算流程。

### 03_试卷分值对应表.xlsx

试卷题号、满分和课程目标映射表。核心字段：

- `所属课程`
- `所属学期`
- `题号`
- `题型`
- `满分`
- `课程目标编号`

说明：期末成绩按课程目标汇总时，以此表的题号-课程目标映射为准。

重要：`03_试卷分值对应表.xlsx` 是题号-课程目标-满分配置表，建议人工维护。不要从学生最高分或学生得分明细自动推断题目满分，因为学生最高分不一定等于题目满分。

### 04_试卷题目得分表_长表.xlsx

学生逐题得分长表。核心字段：

- `学号`
- `姓名`
- `课程目标编号` 或 `课程目标`
- `大题`
- `小题号`
- `学生得分`

说明：程序会将 `大题` 和 `小题号` 组合为题号，例如 `一 + 2` 会标准化为 `1-2`。

如果由 `raw_inputs/04_试卷题目得分表_长表.xlsx` 预处理生成，程序会读取原始 sheet `长表`，并将字段 `课程目标` 映射为 `课程目标编号`，输出为：

```text
input_courses/<课程目录>/04_试卷题目得分长表.xlsx
```

标准字段：

- `学号`
- `姓名`
- `课程目标编号`
- `大题`
- `小题号`
- `学生得分`

### raw_inputs 原始成绩表预处理

当前支持两个原始文件：

```text
raw_inputs/副本材料成型技术基础-课程目标达成情况评价表.xlsx
raw_inputs/04_试卷题目得分表_长表.xlsx
```

其中：

- `副本材料成型技术基础-课程目标达成情况评价表.xlsx` 使用 sheet `达成情况评价`，学生数据从第 7 行开始。
- 前 3 列为 `序号`、`学号`、`姓名`。
- 课程目标1和课程目标2各包含 `平时作业`、`实验操作`、`实验报告`、`课堂表现`、`考试试卷`、`达成度`。
- 预处理后转换为“每个学生 × 每个课程目标一行”的长表。
- 会跳过空学号、空姓名和 `平均值` 等汇总行。
- 学号会统一转为字符串，避免科学计数法或丢失前导零。

新的标准成绩表预处理入口为：

```bash
python -m src.main --course <课程目录名> --prepare-scores
```

当前支持以下数据准备场景：

- 无试卷课程已有分项成绩：例如 `过程检查成绩`、`阶段答辩成绩`、`课程报告成绩`，可直接转换为标准长表。
- 无试卷课程只有总评成绩：按 `01_课程主数据表.xlsx` 中的课程目标权重生成估算拆分。
- 试卷型课程已有平时分项成绩：优先使用真实分项成绩。
- 试卷型课程只有平时成绩总分：按平时考核项目满分比例生成估算拆分；试卷部分仍由 03 和 04 精确计算。

标准 `02_学生成绩表.xlsx` 会保留 `数据来源说明` 字段，常见取值包括：

- `真实分项成绩`
- `按课程目标权重拆分`
- `随机扰动拆分，仅用于测试`

注意：估算拆分数据不得作为真实分项成绩使用，不能把估算数据伪装成真实数据。

每次生成标准 02 时，会同步生成 `02_学生成绩表_生成核查报告.xlsx`。核查报告包括：

- 学生人数
- 是否存在缺失学号
- 是否存在重复学号
- 学号是否按字符串读取
- 课程目标编号是否统一
- 每个课程目标人数是否一致
- 是否存在空成绩
- 是否存在超出满分的成绩
- 原始总评成绩与转换后复核总评成绩的误差
- 数据来源说明

### 07_历年课程目标达成值.xlsx

用于生成“3年课程目标达成度对比图”。优先读取 sheet `历年汇总`，如果不存在，则读取第一个 sheet。

推荐字段：

- `学年`
- `课程目标编号`
- `综合平均达成值`

当前也兼容以下字段：

- `年级` 等同于 `学年`
- `达成值` 等同于 `综合平均达成值`

### 08_达成报告草稿.docx 或报告模板

如果使用模板化报告流程，可放置 Word 模板或草稿文件。当前新版 `report_exporter.py` 采用 python-docx 直接生成 `07_达成度报告草稿.docx`，不会强依赖模板文件。

## 7. 输出文件说明

### 05_课程目标结果表.xlsx

包含：

- `学生课程目标结果表`
- `课程目标结果表`

核心结果包括每个学生每个课程目标的过程性贡献、期末贡献、综合达成值，以及每个课程目标的综合平均达成值。

### 06_达成度报告分析底表.xlsx

包含：

- `分析底表`
- `课程目标汇总表`
- `学生个体达成度表`
- `未达成学生清单`
- `达成度分布统计表`
- `图表数据表`

其中 `课程目标汇总表` 使用合格阈值 `0.60` 判断是否达成。

### 07_达成度报告草稿.docx

自动生成 Word 报告草稿，包含：

- 课程基本信息
- 课程目标及支撑关系
- 课程目标达成度评价方法
- 课程目标达成度结果
- 持续改进报告图表
- 学生个体达成情况分析
- 存在问题
- 持续改进措施
- 结论

### figures/

自动生成 3 张 PNG 图表：

```text
figures/01_三年课程目标达成度对比.png
figures/02_课程目标1达成度散点图.png
figures/03_课程目标2达成度散点图.png
```

## 8. Word 模板占位符说明

当前主流程由 `src/report_exporter.py` 直接生成 Word 报告，并在“持续改进报告图表”章节中插入图片。

如果后续切换为模板替换方式，建议在模板中保留以下图表占位符：

```text
{{图1_三年课程目标达成度对比}}
{{图2_课程目标1达成度散点图}}
{{图3_课程目标2达成度散点图}}
```

占位符与当前图表文件对应关系：

- `{{图1_三年课程目标达成度对比}}` 对应 `01_三年课程目标达成度对比.png`
- `{{图2_课程目标1达成度散点图}}` 对应 `02_课程目标1达成度散点图.png`
- `{{图3_课程目标2达成度散点图}}` 对应 `03_课程目标2达成度散点图.png`

## 9. 课程目标编号兼容规则

程序会将课程目标编号统一标准化为 `课程目标N` 格式。

当前兼容示例：

```text
1        -> 课程目标1
目标1    -> 课程目标1
CO1      -> 课程目标1
课程目标1 -> 课程目标1
```

注意：不同输入表中的课程目标编号应表达同一含义，否则会影响 merge 和统计结果。

## 10. 常见问题

### 课程目标编号不一致

现象：

- 过程性成绩或期末成绩合并后为空。
- 某些课程目标达成值异常为 0 或缺失。

处理：

- 检查 `01`、`02`、`03`、`04` 中课程目标编号是否一致。
- 使用 `DEBUG=True` 查看读取后和标准化后的唯一值。
- 配置位置：`src/config.py`。

### 题号匹配失败

现象：

- 期末成绩无法按课程目标正确汇总。
- debug 输出中“期末试卷明细未匹配到 03 题号数量”大于 0。

处理：

- 检查 `03_试卷分值对应表.xlsx` 的 `题号`。
- 检查 `04_试卷题目得分表_长表.xlsx` 的 `大题`、`小题号`。
- 当前程序会把中文大题 `一、二、三...` 转为 `1、2、3...` 并组合为 `1-2` 形式。

### 不要用学生最高分推断题目满分

`03_试卷分值对应表.xlsx` 应作为人工维护的配置表，明确每道题对应的课程目标和满分。学生得分表只反映学生实际得分，不能保证最高分等于题目满分，因此不能用学生最高分自动反推满分。

### python-docx 缺失

现象：

```text
ModuleNotFoundError: No module named 'docx'
```

处理：

```bash
python -m pip install python-docx
```

或重新安装全部依赖：

```bash
python -m pip install -r requirement.txt
```

### matplotlib 中文字体问题

现象：

- 图表中文显示为方框。
- 运行时出现字体相关 warning。

处理：

- Windows 环境通常可使用 `Microsoft YaHei` 或 `SimHei`。
- 程序已在 `chart_exporter.py` 中配置常见中文字体候选。
- 如仍有问题，请在系统中安装中文字体，或调整 `plt.rcParams["font.sans-serif"]`。

### 历年达成值文件 sheet 或列名不一致

推荐使用：

```text
sheet: 历年汇总
字段: 学年, 课程目标编号, 综合平均达成值
```

当前兼容：

```text
sheet: 第一个 sheet
字段: 年级, 课程目标编号, 达成值
```
